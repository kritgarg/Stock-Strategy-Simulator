# Contributing to Stock Strategy Simulator

Thank you for your interest in contributing to the Stock Strategy Simulator! We welcome contributions from the community to help improve this project.

## ü§ù How to Contribute

We welcome contributions in the following forms:
-   **Bug Fixes:** Identifying and resolving issues in the code.
-   **Feature Enhancements:** Adding new indicators, metrics, or UI improvements.
-   **Documentation:** Improving the README or adding inline comments.

### Contribution Rules

1.  **Code of Conduct:** Please be respectful and professional in all interactions.
2.  **No Financial Advice:** Do not add features that make specific financial recommendations or promises of profit.
3.  **Clean Code:** Follow PEP 8 standards for Python code.
4.  **Testing:** Ensure your changes do not break existing functionality. Run the app locally before submitting.

## üöÄ Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally:
    ```bash
    git clone https://github.com/YOUR_USERNAME/Stock-Strategy-Simulator.git
    ```
3.  **Create a new branch** for your feature or fix:
    ```bash
    git checkout -b feature/amazing-feature
    ```
4.  **Make your changes** and commit them with clear messages:
    ```bash
    git commit -m "feat: Add EMA indicator support"
    ```
5.  **Push to your fork**:
    ```bash
    git push origin feature/amazing-feature
    ```
6.  **Open a Pull Request** (PR) to the `master` branch of the original repository.

## üìù Code Style

-   Use meaningful variable names (e.g., `closing_price` instead of `cp`).
-   Add comments for complex logic blocks.
-   Keep functions small and focused on a single task.
